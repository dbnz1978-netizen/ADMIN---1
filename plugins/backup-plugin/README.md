# Плагин резервного копирования

## Описание

Система резервного копирования сайта с возможностью выбора таблиц базы данных и папок для архивации. Включает PHP-установщик для восстановления сайта на новом хостинге.

## Установка

1. Откройте админ-панель
2. Перейдите в раздел "Плагины"
3. Найдите "Система резервного копирования"
4. Нажмите "Установить"
5. Включите плагин

## Использование

### Создание резервной копии

1. В меню выберите "Резервное копирование" → "Создать резервную копию"
2. Выберите нужные таблицы базы данных (по умолчанию все)
3. Выберите нужные папки (по умолчанию все)
4. Нажмите "Создать резервную копию"
5. Скачайте созданный архив

### Содержимое архива

- `database.sql` - дамп базы данных
- `files/` - выбранные папки и файлы
- `install.php` - установщик с Bootstrap интерфейсом
- `README.txt` - инструкция по установке

### Восстановление сайта

1. Загрузите архив на новый сервер
2. Распакуйте архив
3. Откройте `install.php` в браузере
4. Введите данные для подключения к базе данных
5. Нажмите "Установить сайт"
6. После установки удалите `install.php`

## Возможности

- ✅ Выбор таблиц базы данных
- ✅ Выбор папок для архивации
- ✅ Автоматическое включение папки admin
- ✅ PHP установщик с Bootstrap интерфейсом
- ✅ README с инструкциями
- ✅ CSRF защита
- ✅ Валидация данных
- ✅ Защита от SQL инъекций
- ✅ Защита от path traversal атак

## Требования

- PHP 7.4+
- Расширения PHP: PDO, MySQL, ZIP
- Права на запись в `/admin/backups`

## Безопасность

- Доступ только для администраторов
- CSRF защита всех форм
- Валидация всех входных данных
- Защита от SQL инъекций
- Защита от path traversal атак

## Ограничения

- Размер архива ограничен настройками PHP (memory_limit, max_execution_time)
- Требуется установленное расширение ZIP
- Символические ссылки копируются как файлы
